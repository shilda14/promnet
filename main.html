Definisi

Git adalah sistem pengendalian versi terdistribusi yang digunakan untuk mengelola perubahan dalam kode sumber proyek perangkat lunak. Ini memungkinkan pengembang untuk melacak, menyimpan, dan berkolaborasi dalam pengembangan perangkat lunak dengan efisien. Git dirancang untuk mengatasi tantangan dalam pengelolaan versi perangkat lunak dan memungkinkan pemeliharaan riwayat perubahan, pengembangan bersama tim, serta penggabungan kode dengan aman. Git memungkinkan pemisahan proyek ke dalam cabang-cabang pengembangan yang berbeda, yang mempermudah pengembangan fitur baru tanpa mengganggu kode yang sudah ada.

Konsep Dasar

1. Snapshot

Git berbasis pada konsep snapshot. Setiap kali Anda melakukan commit, Git membuat salinan lengkap dari semua file dalam repositori pada saat itu. Ini berbeda dengan sistem pengendalian versi lain yang mungkin hanya menyimpan perubahan (diff).

2. Branch

Branch adalah cabang pengembangan yang memungkinkan Anda bekerja pada fitur atau perbaikan tertentu tanpa mempengaruhi kode di branch utama (biasanya disebut master atau main).

3. Commit

Commit adalah tindakan untuk menyimpan snapshot saat ini ke repositori Git. Setiap commit memiliki pesan yang menjelaskan perubahan yang dilakukan.

4. Remote

Remote adalah repositori Git yang berada di server jarak jauh. Ini memungkinkan kolaborasi tim dan sinkronisasi antara repositori lokal dan repositori di server.

5. Clone

Mengklon adalah tindakan membuat salinan repositori Git dari server ke komputer lokal Anda.

6. Pull

Pull adalah tindakan mengambil perubahan dari repositori remote dan menggabungkannya dengan repositori lokal Anda.

7. Push

Push adalah tindakan mengunggah perubahan dari repositori lokal Anda ke repositori remote.

Manfaat

- Melacak perubahan dalam kode sumber.

- Memungkinkan kolaborasi tim dengan aman.

- Membantu mengatasi konflik saat penggabungan kode.

- Membantu dalam pengelolaan versi perangkat lunak.
